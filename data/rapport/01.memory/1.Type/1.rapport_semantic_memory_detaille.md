# Rapport détaillé — Brique `SemanticMemory`

## 1. Objectif de la brique

`SemanticMemory` représente la mémoire **long terme structurée** des agents.

Elle stocke :
- des faits durables
- du contexte projet
- des décisions importantes
- des préférences
- des connaissances réutilisables

👉 Contrairement à `ConversationMemory`, cette mémoire est :
- persistante
- structurée
- réutilisable entre sessions

---

## 2. Décisions d’architecture retenues

### 2.1 Nature des données
On stocke des **unités de connaissance atomiques** appelées :

👉 `MemoryEntry`

Chaque entrée représente un fait exploitable indépendamment.

---

### 2.2 Scope

On introduit les scopes suivants :

```text
- AGENT
- SHARED
- PROJECT
- SYSTEM
```

👉 Permet :
- isolation
- mutualisation
- extensibilité future (multi-user)

---

### 2.3 Typage des données

```text
- FACT
- DECISION
- CONTEXT
- PREFERENCE
- KNOWLEDGE
```

👉 Important pour :
- filtrage
- priorisation
- évolution future

---

### 2.4 Stockage V1

👉 Base relationnelle (PostgreSQL recommandé)

Pourquoi :
- filtrage structuré
- robustesse
- simplicité
- auditabilité

---

### 2.5 Ce qu’on ne fait PAS en V1

- pas d’embeddings
- pas de vector DB
- pas de recherche sémantique complexe
- pas de ML

👉 On reste :
- simple
- déterministe
- contrôlable

---

## 3. Risques et arbitrages

### 3.1 Stockage de bruit

Si on stocke tout :
- pollution
- baisse de qualité
- coûts inutiles

👉 Arbitrage :
- pipeline de validation
- importance minimale
- tagging obligatoire

---

### 3.2 Mauvaise classification

Un mauvais type/scope rend la mémoire inutilisable.

👉 Arbitrage :
- typage strict
- validation côté service
- possibilité override manuel

---

### 3.3 Explosion volumétrique

👉 Arbitrage :
- TTL possible
- archivage
- filtrage strict au retrieval

---

## 4. Périmètre V1

## À implémenter
- CRUD mémoire
- stockage SQL
- typage + scope
- importance
- tags
- recherche simple (filtre + texte LIKE)
- scoring simple (importance + usage + récence)

## À ne pas implémenter
- vector search
- embeddings
- clustering
- ML
- auto-learning avancé

```java
    OBSOLETE

```

---

## 5.2 Entité principale

```java
@Entity
@Table(name = "semantic_memory")
public class MemoryEntry {

    @Id
    @GeneratedValue
    private Long id;

    private String agentId;

    @Enumerated(EnumType.STRING)
    private MemoryScope scope;

    private String scopeId; // ex: projectId

    @Enumerated(EnumType.STRING)
    private MemoryType type;

    @Column(length = 5000)
    private String content;

    private double importance;

    private int usageCount;

    private Instant lastAccessedAt;

    private Instant createdAt;

    private Instant updatedAt;

    @Enumerated(EnumType.STRING)
    private MemoryStatus status;

    @ElementCollection
    private Set<String> tags;
}
```

---

## 5.3 Interfaces métier

```java
public interface SemanticMemoryRepository {
```

```java
public interface SemanticMemoryService {
```

## 6. Algorithme de scoring

```text
score = importance
      + (usageCount * 0.5)
      + recencyBoost
```

avec :

```text
recencyBoost = 1 / (1 + age_in_days)
```
