# Rapport détaillé — Brique `Memory Scoring`

## 1. Objectif de la brique

`Memory Scoring` est le mécanisme qui permet de **prioriser les mémoires** afin de sélectionner celles à injecter dans le contexte du LLM.

👉 Il répond à une problématique clé :
> Toutes les mémoires ne se valent pas, il faut choisir les bonnes.

---

## 2. Principe fondamental

Chaque mémoire possède un **score dynamique** basé sur :

- importance (statique)
- fréquence d’utilisation (usage)
- récence (temps)

👉 Formule V1 :

```text
score = importance
      + (usageCount * weightUsage)
      + recencyBoost
```

---

## 3. Décisions d’architecture retenues

### 3.1 Score NON stocké

👉 Le score est **calculé dynamiquement**, jamais stocké en base.

Pourquoi :
- évite incohérences
- recalcul toujours à jour
- adaptable facilement

---

### 3.2 Champs nécessaires

Chaque mémoire doit exposer :

```text
importance
usageCount
lastAccessedAt
createdAt
```

---

### 3.3 Centralisation

👉 Un seul service :

```text
MemoryScoringService
```

Utilisé par :
- SemanticMemory
- EpisodicMemory (futur)
- ContextAssembler

---

## 6. Modèle Java

## 6.1 Interface

```java

    double getImportance();

    int getUsageCount();

    Instant getLastAccessedAt();

    Instant getCreatedAt();
}
```

---

## 6.2 Service

```java
public interface MemoryScoringService {

    double computeScore(ScorableMemory memory);

}
```

---

## 6.3 Implémentation

```java
public class DefaultMemoryScoringService implements MemoryScoringService {

    private final double usageWeight = 0.5;

    @Override
    public double computeScore(ScorableMemory memory) {

        double importance = memory.getImportance();
        double usage = memory.getUsageCount() * usageWeight;
        double recency = computeRecency(memory.getLastAccessedAt());

        return importance + usage + recency;
    }

    private double computeRecency(Instant lastAccessedAt) {
        long days = Duration.between(lastAccessedAt, Instant.now()).toDays();
        return 1.0 / (1 + days);
    }
}
```

---

## 7. Algorithme détaillé

## 7.1 Recency

```text
recency = 1 / (1 + age_in_days)
```

- récent → proche de 1
- ancien → proche de 0

---

## 7.2 Usage

```text
usageScore = usageCount * weight
```

Option V1 :
- weight = 0.5

---

## 7.3 Score final

```text
score = importance + usageScore + recency
```

---

## 7.4 Tri

```text
order by score desc
take top N
```

---

## 8. Pipeline

## 8.1 Retrieval
