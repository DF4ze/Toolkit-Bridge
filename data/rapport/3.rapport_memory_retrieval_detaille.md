# Rapport détaillé — Brique `Memory Retrieval`

## 1. Objectif de la brique

`Memory Retrieval` est responsable de **sélectionner les mémoires pertinentes** à injecter dans le contexte du LLM.

👉 C’est la brique qui transforme :
- un grand volume de mémoire
👉 en
- un contexte utile, compact et pertinent

---

## 2. Rôle dans l’architecture

Pipeline global :

```text
User input
    → Intent (optionnel V2)
    → Memory Retrieval
    → ContextAssembler
    → LLM
```

👉 Elle agit comme un **filtre intelligent**

---

## 3. Décisions d’architecture retenues

### 3.1 Séparation des responsabilités

## 6. Modèle Java

## 6.1 Interface

```java
public interface MemoryRetriever {

    List<MemoryEntry> retrieve(MemoryQuery query);

}
```

---

## 6.2 Query

```java
public record MemoryQuery(
        String agentId,
        String projectId,
        String textQuery,
        Set<MemoryScope> scopes,
        Set<MemoryType> types,
        int limit
) {
}
```

---

## 6.3 Implémentation

```java
public class DefaultMemoryRetriever implements MemoryRetriever {

    private final SemanticMemoryRepository repository;
    private final MemoryScoringService scoringService;

    public DefaultMemoryRetriever(
            SemanticMemoryRepository repository,
            MemoryScoringService scoringService
    ) {
        this.repository = repository;
        this.scoringService = scoringService;
    }

    @Override
    public List<MemoryEntry> retrieve(MemoryQuery query) {

        List<MemoryEntry> candidates = repository.search(query.agentId(), query.textQuery());

        return candidates.stream()
                .filter(m -> query.scopes().contains(m.getScope()))
                .filter(m -> query.types().contains(m.getType()))
                .sorted((a, b) -> Double.compare(
                        scoringService.computeScore(b),
                        scoringService.computeScore(a)
                ))
                .limit(query.limit())
                .toList();
    }
}
```

## 7. Algorithme détaillé

## 7.1 Filtrage

```text
WHERE scope IN (...)
AND type IN (...)
AND status = ACTIVE
```

---

## 7.2 Recherche texte

```text
content LIKE %query%
OR tags LIKE %query%
```

---

## 7.3 Scoring

```text
score = importance + usage + recency
```

---

## 7.4 Tri

```text
ORDER BY score DESC
```

---

## 7.5 Limitation

```text
LIMIT N
```

---

## 8. Pipeline

