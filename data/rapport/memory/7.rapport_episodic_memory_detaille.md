# Rapport détaillé — Brique `EpisodicMemory`

## 1. Objectif de la brique

`EpisodicMemory` représente la mémoire **des actions et de leurs résultats**.

Elle stocke :
- ce que l’agent a fait
- dans quel contexte
- avec quel résultat
- avec quelles conséquences

👉 C’est la brique clé pour :
- audit
- debug
- amélioration continue
- apprentissage futur

---

## 2. Positionnement dans l’architecture

Contrairement :
- à `ConversationMemory` → court terme
- à `SemanticMemory` → faits durables
- à `RuleMemory` → comportement

👉 `EpisodicMemory` = **historique brut des événements**

C’est une mémoire :
- volumineuse
- append-only
- orientée “log enrichi”

---

## 3. Décisions d’architecture retenues

### 3.1 Nature des données

Chaque élément est un :

👉 `EpisodeEvent`

Un événement représente :
- une action
- un état
- un résultat

---

### 3.2 Types d’événements

```text
- ACTION (ex: exécution tool)
- DECISION (ex: BUY / SELL)
- RESULT (ex: succès / erreur)
- OBSERVATION (ex: état du marché)
- ERROR
```

---

### 3.3 Structure événementielle

Chaque événement contient :

- contexte
- action
- résultat
- metadata

👉 proche d’un event sourcing simplifié

---

### 3.4 Scope

```text
- AGENT
- PROJECT
- SYSTEM
```

---

### 3.5 Stockage V1

👉 Base SQL (table append-only)

Pourquoi :
- audit
- simplicité
- requêtage
- robustesse

---

### 3.6 Ce qu’on ne fait PAS en V1

- pas d’apprentissage automatique
- pas de scoring complexe
- pas de vectorisation
- pas de clustering

---

## 4. Risques et arbitrages

### 4.1 Volumétrie

👉 Arbitrage :
- append-only simple
- archivage futur
- requêtes limitées

---

### 4.2 Difficulté d’exploitation

👉 Arbitrage :
- structure claire
- typage strict
- champs obligatoires

---

### 4.3 Pollution par bruit

👉 Arbitrage :
- niveau de granularité contrôlé
- ne pas logger “tout”
- privilégier événements significatifs

---

## 6. Modèle de données Java

## 6.1 Enums

```java
public enum EpisodeEventType {
    ACTION,
    DECISION,
    RESULT,
    OBSERVATION,
    ERROR
}
```

```java
public enum EpisodeStatus {
    SUCCESS,
    FAILURE,
    UNKNOWN
}
```

---

## 6.2 Entité

```java
@Entity
@Table(name = "episodic_memory")
public class EpisodeEvent {

    @Id
    @GeneratedValue
    private Long id;

    private String agentId;

    private String scopeId; // project, trade, etc.

    @Enumerated(EnumType.STRING)
    private EpisodeEventType type;

    private String action;

    @Column(length = 5000)
    private String details;

    @Enumerated(EnumType.STRING)
    private EpisodeStatus status;

    private double score; // ex: PnL, succès, etc.

    private Instant createdAt;
}
```

---

## 6.3 Interfaces

```java
public interface EpisodicMemoryRepository {
    EpisodeEvent save(EpisodeEvent event);
    List<EpisodeEvent> findByAgent(String agentId);
}
```

```java
public interface EpisodicMemoryService {
    EpisodeEvent record(EpisodeEvent event);
    List<EpisodeEvent> findRecent(String agentId, int limit);
}
```
