- sans réintroduire aucun fallback runtime YAML

⚠️ Important :
- il ne s’agit pas encore d’implémenter la nouvelle vraie Phase 2 de bootstrap complet
- il s’agit uniquement de nettoyer la séparation de responsabilité maintenant

### 2. Nettoyer la configuration runtime LLM

- garde une configuration runtime cohérente avec le modèle DB-first

Pour `deleteByAgentIdentNotIn(...)` :

### 4. Simplifier le code devenu inutilement défensif

Supprime les checks `null` devenus redondants sur les listes chargées depuis le gateway si ces méthodes garantissent déjà une liste non nulle.

### 5. Tests

Ajoute ou ajuste uniquement les tests réellement utiles pour verrouiller :
- que la lecture runtime ne dépend plus d’un seed YAML
