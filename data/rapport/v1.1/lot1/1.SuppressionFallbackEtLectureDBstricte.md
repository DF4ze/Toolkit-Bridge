# 🔹 Phase 1 — Suppression du fallback + lecture DB stricte

## 🎯 Objectif

- Supprimer toute dépendance runtime au YAML pour la configuration métier
- Lire uniquement depuis la DB
- Permettre un démarrage même si la DB est vide

---

## 🧩 Travail à réaliser

### 1. Supprimer le fallback YAML runtime

Dans :
- `AdministrableConfigurationGateway`

Actions :
- supprimer tous les `orElseGet(seed...)`
- ne plus appeler de seed YAML dans les méthodes de lecture
- retourner uniquement les données DB
- retourner vide si absent

⚠️ IMPORTANT :
- le YAML ne doit plus intervenir en runtime métier

---

### 2. Adapter les services consommateurs

Identifier et adapter :
- `AgentDefinitionService`
- `OpenAiLikeConfiguration`
- autres consommateurs

Actions :
- accepter des listes vides
- ne pas lever d’erreur si aucune config

---

### 3. Permettre un démarrage avec DB vide

Actions :
- supprimer ou adapter les validations bloquantes
- notamment :
    - `@NotEmpty` sur Agents YAML
    - autres contraintes empêchant démarrage sans config

---

### 4. Désactiver les comportements destructifs

Dans :
- `AgentAccountInitializer`

Actions :
- supprimer `deleteByAgentIdentNotIn`
- éviter toute suppression automatique en DB

---

## ⚠️ Contraintes

- Ne pas toucher au bootstrap
- Ne pas implémenter de seed ici
- Ne pas modifier la structure DB
- Ne pas introduire de reload runtime

---

## ✅ Livrables attendus

- fallback supprimé
- démarrage OK avec DB vide
- runtime sans dépendance YAML
