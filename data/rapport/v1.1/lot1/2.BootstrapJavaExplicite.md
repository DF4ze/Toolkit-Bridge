# 🔹 Phase 2 — Bootstrap Java explicite (champ par champ)

## 🎯 Objectif

- Introduire un bootstrap propre
- Compléter la DB uniquement sur les champs manquants
- Ne jamais écraser l’existant

---

## 🧩 Travail à réaliser

### 1. Créer un bootstrap dédié

Créer :
- `AdministrableConfigurationBootstrapV2`

Type :
- `ApplicationRunner`

Responsabilité :
- compléter la DB au démarrage

---

### 2. Principe du bootstrap

Pour chaque domaine :
- si absent en DB → créer
- si présent → ne rien modifier

---

### 3. Champs concernés

Bootstrap UNIQUEMENT pour :

- LLM providers
- paramètres techniques migrés vers DB (artifacts, retention…)
- paramètres mémoire (structure initiale)

⚠️ EXCLUS :
- Agents
- Bots Telegram

---

### 4. Granularité

Ne pas faire :
- remplacement complet de liste

Faire :
- injection fine (objet par objet / champ par champ)

---

### 5. Logging

Ajouter logs :

- `BOOTSTRAP: missing -> created`
- `BOOTSTRAP: existing -> skipped`

---

### 6. Ne pas réintroduire de fallback

⚠️ CRITIQUE :

- bootstrap exécuté uniquement au démarrage
- jamais utilisé en runtime

---

## ⚠️ Contraintes

- Ne jamais écraser l’existant
- Ne pas dépendre du YAML en runtime
- YAML utilisé uniquement comme seed au démarrage

---

## ✅ Livrables attendus

- bootstrap propre
- logs explicites
- aucun effet de bord runtime
