### 2. Invariant status / decisionStatus

Objectif:
- garantir cohérence des statuts

Actions:
- lors de `recordDecision` :
    - s’assurer que :
        - `status = decision.status`
        - `decisionStatus = decision.status`
- ajouter test unitaire :
    - vérifier que les deux champs sont identiques après décision

---

### 3. Logging metadata

Objectif:
- éviter perte silencieuse

Actions:

- si JSON > limite :
    - log.warn (message explicite + requestId si possible)
- si erreur de désérialisation :
    - log.warn

⚠️ garder fallback `{}`

---

### 4. Tests fragiles

Objectif:
- rendre les tests robustes

- vérifier :
    - présence de clés
