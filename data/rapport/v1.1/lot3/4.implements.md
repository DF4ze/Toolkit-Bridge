
# Prompt Codex — Implémentation Phase 4 — Rétention minimale

Tu travailles sur le projet Toolkit-Bridge / Marcel.

## 🎯 Objectif

Implémenter une politique de rétention minimale pour éviter
une croissance infinie des données persistées introduites dans le lot 3 :

- traces critiques
- tâches admin

⚠️ IMPORTANT :
- Implémentation simple, robuste, sans sur-architecture
- Aucun système complexe d’archivage
- Aucune refonte des modèles existants

---

## ⚠️ Contraintes absolues

- ❌ Pas de refactor global
- ❌ Pas de système d’archivage
- ❌ Pas de multi-tier storage
- ❌ Pas de logique métier complexe

- ✅ TTL simple
- ✅ purge basée sur timestamp
- ✅ exécution planifiée (scheduler)
- ✅ configuration externalisée

---

## 📌 Règles validées

TTL par défaut :

- traces critiques → 7 jours
- tâches admin → 30 jours
- interventions humaines → PAS de purge (pour l’instant)

Déclenchement :
- tâche planifiée (Spring @Scheduled)

---

## 🧱 Travail à réaliser

### 1. Configuration de rétention

Créer une configuration dédiée, par exemple :

`PersistenceRetentionProperties`

Propriétés attendues :

```properties
toolkit.persistence.retention.traces-days=7
toolkit.persistence.retention.tasks-days=30
toolkit.persistence.retention.enabled=true
toolkit.persistence.retention.cron=0 0 * * * *
````

⚠️ Valeurs par défaut obligatoires

---

### 2. Implémenter les méthodes de purge repository

#### 2.1 Traces critiques

Dans `CriticalAgentTraceRepository` :

* ajouter méthode :

```text
deleteByOccurredAtBefore(Instant cutoff)
```

---

#### 2.2 Tâches admin

Dans `AdminTaskSnapshotRepository` :

* ajouter méthode :

```text
deleteByLastSeenAtBefore(Instant cutoff)
```

⚠️ Ne jamais toucher `firstSeenAt`

---

### 3. Implémenter le service de purge

Créer un service dédié :

`PersistenceRetentionService`

Responsabilités :

* calculer les cutoffs :

    * traces → now - tracesDays
    * tasks → now - tasksDays
* appeler les repositories
* logguer :

    * nombre de lignes supprimées
    * type de données

⚠️ Ne pas purger les interventions humaines

---

### 4. Planification (scheduler)

Créer un scheduler :

`PersistenceRetentionScheduler`

* annoté `@Scheduled(cron = ...)`
* activé uniquement si :

    * `retention.enabled = true`

Fréquence :

* configurable via properties
* défaut : toutes les heures

---

### 5. Logging

Ajouter logs explicites :

```text
Purged X critical traces older than cutoff
Purged Y admin tasks older than cutoff
```

⚠️ Pas de logs bruyants
⚠️ Niveau INFO ou DEBUG selon ton choix (je recommande INFO léger)

---

### 6. Sécurité / robustesse

* ne pas planter si DB vide
* ne pas lancer si TTL <= 0
* protéger contre valeurs absurdes

---

### 7. Tests

#### 7.1 Tests unitaires

* cutoff correctement calculé
* appel repository correct

#### 7.2 Tests intégration

* insertion données anciennes
* exécution purge
* vérification suppression

#### 7.3 Test scheduler (optionnel léger)

* vérifier déclenchement (mock)
