# Prompt Codex — Correction — Isolation des bases de test

## 🎯 Objectif

Supprimer la dépendance à une base de données de test globale persistante
et rendre les tests d’intégration déterministes.

---

## 📌 Contexte

Actuellement :
- `src/test/resources/application.yml` utilise :
    - `jdbc:sqlite:./data/toolkit-bridge-test.db`
- cela crée :
    - état persistant entre runs
    - pollution inter-tests
    - divergences IntelliJ / Maven
    - erreurs non déterministes

---

## ⚠️ Contraintes

- ❌ Pas de refactor global
- ❌ Pas de changement métier
- ❌ Pas de migration H2
- ❌ Pas de changement prod

- ✅ Garder SQLite
- ✅ Isolation par classe de test
- ✅ Impact minimal

---

## 🛠️ Travail à réaliser

### 1. Supprimer la DB test globale

Dans `src/test/resources/application.yml` :

- supprimer la datasource SQLite persistante globale

---

### 2. Isoler les tests SpringBootTest

Pour chaque test utilisant `@SpringBootTest` sans datasource spécifique :

- ajouter :

```java
@SpringBootTest(properties = {
    "spring.datasource.url=jdbc:sqlite:file:./target/test-db-${random.uuid}.db"
})
```

### 3. Vérifier les tests restart
- conserver leur logique actuelle
- mais s’assurer qu’ils utilisent une DB unique
- pas de DB partagée

### 4. Vérifier MemoryScoringServiceIT
- s’assurer qu’il ne dépend plus d’une config persistée précédente
- vérifier que la property test est bien prise en compte
- ou que le seed DB est cohérent

### 5. Vérifier le build
- exécuter tests IntelliJ
- exécuter Maven si possible 
- vérifier disparition des effets de pollution
