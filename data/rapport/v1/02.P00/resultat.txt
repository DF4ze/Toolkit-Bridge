Limites pour la suite (AgentRuntime complet, multi-agent, tasks, bus)
Pas d’objet runtime central explicite (exécution = service + assemblage implicite).
Frontière policy absente.
Contrat orchestrator centré sur AgentDefinition statique, pas sur runtime.
Peu de points d’accroche explicites pour task model / bus inter-agent.
Plan proposé (et appliqué) en 7 étapes:

Introduire objet AgentRuntime + état runtime minimal.
Formaliser rôle/policy dans la définition statique.
Créer contrat AgentPolicy + registry.
Créer AgentRuntimeFactory pour assembler runtime complet.
Basculer contrat orchestrator vers handle(AgentRuntime, AgentRequest).
Adapter runtime service pour construire/déléguer via runtime.
Mettre à jour et compléter les tests.
