Tu vas implémenter la phase 15.B de mon projet “Toolkit-Bridge”.

Contexte :
Le cadre global du projet, la logique d’implémentation incrémentale, ainsi que la phase 15.A déjà réalisée, te sont fournis séparément.
Tu dois t’y conformer strictement et t’appuyer sur le protocole de débat inter-agent déjà introduit, ainsi que sur le modèle d’artefact, le bus, les tâches et les traces existants.

Objectif de cette phase :
commencer la collaboration inter-agent par des scénarios simples, concrets et traçables, afin de mesurer la valeur réelle du débat sans introduire un système trop ambitieux.

Cette phase ne doit pas encore implémenter :
- un grand parlement multi-agent
- des débats avec beaucoup de rôles simultanés
- des mécanismes de vote complexes
- des workflows d’orchestration lourds
- une généralisation prématurée à tous les cas d’usage

Le but est de démarrer petit, utilement et proprement.

Périmètre attendu :

1. Commencer par des scénarios simples
   Je veux que la première mécanique de collaboration inter-agent reste petite.
   On doit commencer avec 2 ou 3 rôles maximum selon ce qui est le plus pertinent dans l’état actuel du projet.

2. Introduire une revue croisée sur un artefact
   Je veux un scénario concret de base de type :
- un agent produit un artefact
- un autre agent le critique selon son domaine
- le résultat est traçable

L’objectif est de donner un premier cas d’usage réel au protocole de débat.

3. Garder le résultat traçable
   Je veux que cette collaboration laisse une trace claire :
- qui a produit
- qui a critiqué
- sur quel artefact
- avec quel lien vers la tâche ou la discussion si pertinent

4. Garder la mécanique petite en V1/V2
   Je veux explicitement éviter :
- un grand système de débat généralisé
- trop de rôles simultanés
- une orchestration trop complexe
- des scénarios abstraits sans gain concret

Le but est d’évaluer le gain réel du dispositif.

5. Préparer l’extension future sans l’implémenter
   Cette phase doit préparer plus tard :
- davantage de scénarios de collaboration
- enrichissement des rôles
- arbitrage
- synthèse plus structurée

Mais sans les implémenter maintenant.

Attendus techniques :
- créer ou adapter les classes/interfaces/services nécessaires
- brancher proprement le scénario simple sur le protocole déjà introduit
- rester cohérent avec artefacts, tâches, bus et traces
- garder le code lisible et testable
- ajouter ou adapter les tests pertinents

Ce que je veux explicitement éviter :
- scénario trop générique ou vide
- débat à trop grande échelle trop tôt
- logique métier dispersée
- collaboration non traçable
- sur-conception

Livrables attendus :
1. code complet de la phase 15.B
2. tests
3. résumé court des choix autour des scénarios simples de collaboration inter-agent
4. liste des fichiers modifiés/créés
5. points laissés ouverts pour les phases suivantes

Avant de coder :
- rappelle brièvement ton plan d’implémentation en 5 à 10 étapes maximum
- puis implémente
