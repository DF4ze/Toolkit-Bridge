Pré-analyse (exécutée avant codage)

Fichiers sensibles identifiés: runtime (AgentRuntime*), traces (AgentTrace*), orchestrateur task, artefacts, config administrable, controllers admin existants.
Points d’architecture sensibles: éviter une API admin géante, ne pas contourner les services métier, ne pas figer une UI avant stabilisation des contrats.
Risques/dette: exposition brute des modèles internes, absence de suivi tâche exploitable (les Task étaient éphémères), fuite potentielle de secrets config.
Plan (8 étapes) appliqué: couche admin dédiée, store tâches borné, endpoints techniques ciblés, DTO stables, vue synthétique backend prête UI, tests.
