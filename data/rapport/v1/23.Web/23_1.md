# Lot 3 — Mise en place du socle UI web Marcel (Spring MVC + Thymeleaf + Bootstrap + jQuery)

Tu es un développeur senior Java Spring Boot.

Tu travailles sur mon projet **Tool-Bridge**, dont l’interface d’administration web porte désormais le nom **Marcel**.

Les étapes précédentes ont déjà été traitées :
- le refacto des endpoints d’administration a été lancé
- la sécurité simple par token + login + filtre a été mise en place

Je veux maintenant démarrer proprement la **couche UI web**.

---

## Objectif de ce lot

Mettre en place le **socle UI** de l’administration web Marcel avec :

- Spring MVC
- Thymeleaf
- Bootstrap 5
- jQuery léger
- un thème visuel “Marcel”
- une base propre, réutilisable, maintenable

⚠️ Important :
Ce lot ne doit **pas** encore construire toute l’interface métier.
On veut d’abord :
- les dépendances
- l’organisation des templates
- les ressources statiques
- la charte graphique
- une première page de démonstration
- un début de cohérence visuelle

---

## Intention visuelle

L’interface doit refléter l’identité **Marcel** :

- nom d’application : **Marcel**
- ambiance : admin moderne, chaleureuse, simple
- inspiration légère : provençale / artisanale / française
- pas kitsch
- pas folklorique excessif
- rester sobre, lisible et professionnel

### Palette souhaitée
Inspirée du logo fourni :
- jaune chaud
- bleu profond
- blanc cassé / crème

### Branding
Le logo fourni doit être intégré comme identité principale.
Je veux qu’il soit réutilisable dans les futures pages, notamment dans le header.

---

## Fichier logo

Le logo existe déjà et doit être intégré dans les ressources statiques du projet.

Utilise le logo fourni comme base du branding.
Choisis un emplacement cohérent dans `static/img/`.

---

## Contraintes fortes

- Pas de React
- Pas de Vue
- Pas de build frontend Node
- Pas de Webpack/Vite/etc.
- Pas de JS complexe
- jQuery seulement pour besoins simples plus tard
- Bootstrap via CDN acceptable
- Code simple, lisible, maintenable
- Pas de sur-engineering

---

## Ce que je veux concrètement

# 1. Dépendances et configuration

Vérifie et mets en place ce qui est nécessaire pour utiliser :

- Spring MVC
- Thymeleaf

Si des dépendances manquent côté Maven, ajoute-les proprement.

Bootstrap et jQuery peuvent être intégrés via CDN dans les templates.

Je veux une base minimale et propre, pas une accumulation de libs.

---

# 2. Organisation des dossiers UI

Mets en place une structure claire pour les vues et les assets.

Je veux au minimum une organisation du genre :

- `src/main/resources/templates/`
- `src/main/resources/templates/fragments/`
- `src/main/resources/templates/layouts/`
- `src/main/resources/templates/admin/`
- `src/main/resources/static/css/`
- `src/main/resources/static/js/`
- `src/main/resources/static/img/`

Adapte si nécessaire, mais garde une structure propre et évolutive.

---

# 3. Intégration du logo

- Place le logo dans les ressources statiques
- Utilise un nom de fichier propre et stable
- Prépare son usage futur dans le header
- Fais en sorte qu’il soit déjà affiché dans la page de démonstration

---

# 4. Création du thème CSS “Marcel”

Créer un fichier CSS dédié, par exemple :
- `static/css/marcel-theme.css`

## Objectif
Centraliser l’identité visuelle de l’application.

## Attendus
Définir des variables CSS ou une structure équivalente pour au minimum :

- couleur principale jaune
- couleur principale bleue
- couleur de fond crème / blanc cassé
- couleurs de texte cohérentes
- quelques espacements / rayons / ombres légères si utile

Exemple de logique attendue :
- `--marcel-yellow`
- `--marcel-blue`
- `--marcel-cream`

Tu peux ajuster les noms si tu proposes mieux, mais je veux quelque chose de clair.

## Ajouter aussi quelques classes utilitaires réutilisables
Par exemple :
- classes de fond
- classes de texte
- classes pour cartes / panels / branding

L’idée est de préparer la suite.

---

# 5. Style global de base

Je veux une base visuelle sobre et propre :

- fond général agréable
- bonne lisibilité
- hiérarchie visuelle simple
- cartes ou containers harmonieux
- boutons cohérents avec la charte
- rendu propre aussi bien desktop que mobile

Ne cherche pas à faire une “landing page marketing”.
On est sur une future **console d’administration**.

# 6. Première page de démonstration

Créer une première page web simple permettant de valider le socle.

Par exemple :
- `/admin/ui-preview`
  ou
- `/ui-preview`

Choisis la route la plus cohérente avec l’existant, idéalement dans la zone protégée si cela a du sens avec la sécurité mise en place.

## Cette page doit afficher :
- le logo
- le nom **Marcel**
- un sous-titre du type administration / console / interface d’admin
- quelques composants de démonstration :
    - un bouton principal
    - un bouton secondaire
    - une card
    - un bloc d’alerte ou d’information
    - un petit tableau simple ou une liste simple
    - un champ de formulaire ou input simple

Objectif :
- vérifier que la charte fonctionne
- vérifier que Bootstrap + Thymeleaf + CSS custom fonctionnent ensemble
- disposer d’une page de référence visuelle

---

# 7. Contrôleur MVC minimal

Créer un contrôleur MVC propre pour servir cette page de démonstration.

Je veux quelque chose de simple, par exemple :
- `UiPreviewController`
  ou nom équivalent

Pas de logique métier.
Juste le strict nécessaire pour retourner la vue.
