# Lot 4 — Layout d’administration Marcel + navigation + dashboard

Tu es un développeur senior Java Spring Boot.

Tu travailles sur mon projet **Tool-Bridge**, dont l’interface d’administration web s’appelle désormais **Marcel**.

Le lot précédent a déjà posé :
- le socle UI web
- Thymeleaf
- Bootstrap
- jQuery léger
- le thème visuel Marcel
- le logo
- une page de preview UI

Je veux maintenant construire la **vraie base d’interface d’administration** :
- layout global
- navigation
- fragments réutilisables
- dashboard d’accueil

⚠️ Important :
Dans ce lot, je veux **la structure UI d’admin**, mais pas encore les pages métier complètes ni les CRUD LLM / Bots / Agents.

---

## Objectif de ce lot

Créer un **layout d’administration réutilisable**, responsive, cohérent avec le branding Marcel, avec :

- un header
- une navigation
- une zone de contenu
- un footer léger
- un dashboard d’accueil
- une structure qui servira de base à toutes les futures pages admin

---

## Intention UX / Design

L’interface doit être :
- propre
- lisible
- sobre
- chaleureuse
- un peu “Marcel”
- légèrement inspirée de l’esprit provençal / artisanal
- mais rester une **console d’administration**

Je veux éviter :
- le look “site marketing”
- le look trop froid / technique
- le look kitsch

Le thème déjà mis en place doit être réutilisé et consolidé.

---

## Contraintes fortes

- Spring MVC + Thymeleaf
- Bootstrap 5
- jQuery léger uniquement si nécessaire
- pas de framework JS complexe
- pas de SPA
- pas de React/Vue
- pas de build frontend Node
- pas de sur-engineering

---

# 1. Mettre en place un vrai layout admin réutilisable

Créer une structure Thymeleaf propre pour les pages d’administration.

Je veux un layout global, par exemple dans :
- `templates/layouts/admin-layout.html`

Adapte si tu préfères une autre convention, mais garde quelque chose de clair.

## Ce layout doit permettre d’avoir :
- un header commun
- une navigation commune
- une zone principale de contenu
- un footer léger
- une intégration simple des futures pages

Je veux éviter les duplications de structure HTML dans toutes les pages.

Si tu choisis une approche avec fragments Thymeleaf standards, c’est très bien.
Si tu veux utiliser un layout dialect Thymeleaf, fais-le seulement si c’est vraiment pertinent et léger.
Par défaut, préfère la solution la plus simple.

---

# 2. Créer les fragments réutilisables

Créer des fragments dédiés, par exemple :
- `fragments/header.html`
- `fragments/sidebar.html` ou `fragments/navigation.html`
- `fragments/footer.html`
- éventuellement `fragments/flash-messages.html` si pertinent

## Header
Le header doit contenir :
- le logo Marcel
- le nom “Marcel”
- éventuellement un sous-titre discret du type “Administration”
- un bouton ou lien de logout si l’existant de sécurité le permet déjà

Le header doit être sobre, propre, cohérent avec la charte.

## Navigation
Je veux une navigation d’admin claire, pensée pour les futurs écrans.

Entrées attendues :
- Dashboard
- LLM
- Bots Telegram
- Agents
- Technique

Pour l’instant, certaines routes peuvent pointer vers des pages placeholder si nécessaire, mais le menu doit être prêt.

## Footer
Footer léger, discret, sans surcharge.

---

# 3. Gérer le responsive proprement

Je veux une navigation utilisable :
- sur desktop
- sur mobile

Tu peux choisir l’une des deux approches :
- sidebar sur desktop + version collapsible / topbar mobile
- ou top navigation responsive bien pensée

Je te laisse choisir la meilleure solution pour une admin Thymeleaf + Bootstrap simple.

## Important
Le résultat doit être :
- lisible sur téléphone
- agréable sur desktop
- sans JS complexe

---

# 4. Créer le dashboard d’accueil

Créer une page :
- `templates/admin/dashboard.html`

Servie par :
- `GET /admin`

Je veux que `/admin` devienne la vraie entrée principale de l’interface d’administration.

## Contenu attendu du dashboard

Le dashboard doit être simple mais propre.

Afficher au minimum :
- un titre / bienvenue
- quelques cards de synthèse
- une petite zone “raccourcis”
- éventuellement une petite zone “état système” ou “aperçu”

### Cards de synthèse
Pour l’instant, tu peux utiliser des données mockées ou minimales si les services réels ne sont pas encore prêts.

Exemples de blocs utiles :
- nombre d’agents
- nombre de bots Telegram
- nombre de tâches récentes
- état de l’administration technique

Je veux surtout une structure UI crédible et réutilisable.

### Zone raccourcis
Prévoir des accès rapides vers :
- LLM
- Bots Telegram
- Agents
- Technique

Même si les pages complètes arrivent plus tard.

# 5. Contrôleur MVC dédié

- `AdminPageController`
  ou nom équivalent

## Attendus
