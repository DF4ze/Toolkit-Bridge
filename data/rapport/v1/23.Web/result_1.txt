après relecture :
Revue Critique (phase UI Marcel)
Findings (avant correction, triés par impact)

[Moyen] API de layout un peu surconçue: le paramètre pageScripts imposait du boilerplate vide dans chaque vue et augmentait le risque d’oubli sur les futures pages (admin-layout.html:3, index.html:3, ui-preview.html:3).
[Moyen] Fragments Thymeleaf trop “document”: les fragments étaient initialement structurés comme pages HTML complètes, ce qui est moins lisible et moins net architecturalement pour des briques réutilisables.
[Faible à moyen] Tests UI peu robustes: le test initial du contrôleur vérifiait seulement un retour de String; pas de vérification HTTP/mapping réel.
[Faible] Couverture sécurité de la nouvelle route non explicite: /admin/ui-preview était protégé par convention (/admin/**), mais sans test dédié.
Corrections pertinentes appliquées

Simplification du layout: suppression de pageScripts pour garder une API minimale et stable (admin-layout.html:3, admin-layout.html:18).
Nettoyage des fragments en vrais blocs réutilisables:
head.html:1
admin-header.html:1
scripts.html:1
Test contrôleur rendu utile (MockMvc standalone, vérifie route + vue):
UiPreviewControllerTest.java:14
Test sécurité explicite ajouté pour /admin/ui-preview:
AdminAuthFilterTest.java:35
Vérification architecture demandée

Séparation configuration/runtime: OK. Aucun mélange; runtime UI reste dans templates/static, aucune config métier/système modifiée.
Qualité du modèle implémenté: OK pour ce lot. Contrôleur MVC minimal, pas de logique métier déplacée.
Découplage orchestrator/mémoire/tooling/policy/workspace: OK. Aucun couplage introduit; ces sous-systèmes ne sont pas touchés.
Logique ad hoc/spécifique: faible. Pas de JS complexe, thème centralisé.
Cohérence de nommage/lisibilité: bonne pour le périmètre UI.
Tests utiles: améliorés (mapping MVC + protection de route).
Dette technique introduite: limitée.
Dette restante (non bloquante, non modifiée pour rester dans le périmètre)

login.html conserve un style inline séparé du thème Marcel centralisé.
Dépendance CDN (Bootstrap/jQuery/Google Fonts): acceptable pour ce lot, mais à réévaluer si mode offline/CSP strict est prévu.
